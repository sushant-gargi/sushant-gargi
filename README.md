### 🤖 [AI-Driven Code Generation SaaS Platform](https://github.com/sushant-gargi/lovable-clone)

*Describe an app in plain English and get back a live, running React app at its own throwaway URL.*

![Java](https://img.shields.io/badge/Java-ED8B00?style=for-the-badge&logo=openjdk&logoColor=white)
![Spring Boot](https://img.shields.io/badge/Spring_Boot-6DB33F?style=for-the-badge&logo=springboot&logoColor=white)
![Spring AI](https://img.shields.io/badge/Spring_AI-6DB33F?style=for-the-badge&logo=spring&logoColor=white)
![PostgreSQL](https://img.shields.io/badge/PostgreSQL-4169E1?style=for-the-badge&logo=postgresql&logoColor=white)
![Kubernetes](https://img.shields.io/badge/Kubernetes-326CE5?style=for-the-badge&logo=kubernetes&logoColor=white)
![Redis](https://img.shields.io/badge/Redis-DC382D?style=for-the-badge&logo=redis&logoColor=white)
![Stripe](https://img.shields.io/badge/Stripe-635BFF?style=for-the-badge&logo=stripe&logoColor=white)

Users describe a change in a chat panel, and the backend streams the request to an LLM through Spring AI, grounded in the project's real file tree by a custom advisor and an agentic `read_files` tool that lets the model inspect existing code before it edits anything. Hitting Deploy claims a pod from a self-built Kubernetes runner pool, syncs it from MinIO, and routes it live through a Redis-backed wildcard reverse proxy. It's a real multi-tenant platform: RBAC, Stripe billing, and per-tenant network isolation, not a single-user demo.

<details>
<summary><strong>Engineering highlights</strong></summary>
<br/>

- Built an agentic loop on Spring AI's `ChatClient`: a `FileTreeContextAdvisor` injects live project state, a `read_files` tool lets the model inspect code before editing, and a custom parser converts the streamed response into typed file writes and chat events
- Ephemeral preview infrastructure: idle pods are claimed through a label mutation instead of cold provisioning, isolated the instant they're claimed by a Kubernetes `NetworkPolicy`, and served at `project-{id}.app.domain.com` through a hand-rolled Node.js reverse proxy backed by Redis
- Full Stripe subscription lifecycle: Checkout, Billing Portal, and idempotent webhook handling across activation, upgrades, cancellation, and dunning
- Fine-grained RBAC (`OWNER` / `EDITOR` / `VIEWER`) enforced through a custom SpEL permission bean rather than Spring Security's default authority model
- Architected for 10K+ concurrent streaming sessions with sub-200ms token latency and sub-2s cold-start previews

</details>

### 📦 [eCommerce Microservices Platform](https://github.com/sushant-gargi/eCommerce-microservices)

*Six independently deployable Spring Boot services, wired together with Kafka instead of point-to-point calls.*

![Java](https://img.shields.io/badge/Java-ED8B00?style=for-the-badge&logo=openjdk&logoColor=white)
![Spring Boot](https://img.shields.io/badge/Spring_Boot-6DB33F?style=for-the-badge&logo=springboot&logoColor=white)
![Apache Kafka](https://img.shields.io/badge/Apache_Kafka-000000?style=for-the-badge&logo=apachekafka&logoColor=white)
![PostgreSQL](https://img.shields.io/badge/PostgreSQL-4169E1?style=for-the-badge&logo=postgresql&logoColor=white)
![Docker](https://img.shields.io/badge/Docker-2496ED?style=for-the-badge&logo=docker&logoColor=white)

Placing an order synchronously reduces stock and books shipping over OpenFeign, then publishes an Avro-encoded event to Kafka. Inventory and Notification react to it asynchronously, with schema contracts enforced end to end by Confluent Schema Registry. Six services, six databases, one Docker Compose file. Config lives in a separate [Git-backed config server](https://github.com/sushant-gargi/ecommerce-config-server).

<details>
<summary><strong>Engineering highlights</strong></summary>
<br/>

- Sync and async hybrid: stock reduction is synchronous (Feign) for immediate consistency, status propagation is asynchronous (Kafka) for loose coupling and resilience
- Resilience4j layered on every inventory call: a 100 req/s rate limiter, a 3-attempt retry, and a circuit breaker that opens at 50% failure across a 10-call sliding window
- Full distributed tracing: every log line carries a traceId and spanId via Micrometer, exported to Zipkin at 100% sampling for a complete Gateway → Order → Inventory → Shipping request graph
- Netflix Eureka discovery and a Git-backed Spring Cloud Config Server with live `@RefreshScope` updates, so config changes need no restart

</details>

### 🎙️ [AI Mock Interview Platform](https://github.com/sushant-gargi/ai-interview)

*Schedule an interview and the platform runs the whole thing itself: adaptive AI questioning, live voice, and a scored PDF report at the end.*

![Java](https://img.shields.io/badge/Java-ED8B00?style=for-the-badge&logo=openjdk&logoColor=white)
![Spring Boot](https://img.shields.io/badge/Spring_Boot-6DB33F?style=for-the-badge&logo=springboot&logoColor=white)
![Spring AI](https://img.shields.io/badge/Spring_AI-6DB33F?style=for-the-badge&logo=spring&logoColor=white)
![PostgreSQL](https://img.shields.io/badge/PostgreSQL-4169E1?style=for-the-badge&logo=postgresql&logoColor=white)
![JWT](https://img.shields.io/badge/JWT-000000?style=for-the-badge&logo=jsonwebtokens&logoColor=white)

A recruiter schedules a session, the candidate uploads a resume and talks to the AI interviewer by voice, and a structured evaluation report generates itself the moment the interview ends, however it ends. No human intervention needed after scheduling.

<details>
<summary><strong>Engineering highlights</strong></summary>
<br/>

- Voice-native interviewing: the Web Speech API captures spoken answers, Kokoro TTS speaks the AI's replies back as audio, and a transient `minutesRemaining` system message (never persisted) keeps the interviewer's pacing accurate on every turn
- Idempotent report generation across three independent trigger paths (the candidate ending the call, a no-show scheduler, and a disconnect scheduler), all converging safely on one report through a pre-write existence check
- A fully stateless `ChatServiceImpl`: every SYSTEM/USER/ASSISTANT message is persisted to Postgres and replayed on each turn, so any server instance can pick up any request
- Structured, type-safe AI output via Spring AI's `BeanOutputConverter`, so the evaluation is parsed into a typed report instead of pulled out of free text with regex

</details>
